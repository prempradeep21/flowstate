# Mobile SDLC — team skill (sandbox)

Use this skill when questions relate to shipping a mobile app from PRD to App Store.

## Phases

1. **Charter** — PRD, competitive analysis, business case
2. **Discovery** — user research, personas, problem framing
3. **Design** — Figma, wireframes, handoff checklist
4. **Build** — repo, sprint backlog, API spec, architecture
5. **Quality** — test plans, beta feedback, review guidelines
6. **Ship & Learn** — release calendar, KPIs, store submission

## Conventions

- Prefer **Google Drive URLs** (Docs/Sheets) over local Office files when plugging into questions — local `.xlsx` previews on canvas but does not feed AI context.
- **Figma**: paste share links only; no `.fig` upload.
- **One card = one artifact** when generating outputs (Phase 2+).
- **Timeline** for roadmaps; **calendar** for month-grid deadlines.

## Definition of done (mobile)

- iOS and Android parity documented unless explicitly scoped
- Offline/error states covered in design handoff
- App Store / Play metadata and privacy labels listed before ship
