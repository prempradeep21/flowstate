## Color Blind Safe Palette

These neon colors are NOT inherently color-blind safe due to their saturation. If color-blind users are a priority, consider implementing:

1. **Protanopia (Red-Blind)**: Shift cyan/lime palettes, reduce red component
2. **Deuteranopia (Green-Blind)**: Reduce green component, increase blue
3. **Tritanopia (Blue-Blind)**: Shift cyan/magenta to warmer hues

Example alternative for protanopia:
- Cyan → Sky Blue (#00D4FF)
- Lime → Yellow (#FFFF00)
- Magenta → Cyan (#00FFFF)

## Usage Code Examples

### In Tailwind Classes
```tsx
// Text with neon color
className="dark:text-cyan-300"

// Background with opacity
className="dark:bg-cyan-500/35"

// Border with glow
className="dark:border-cyan-400/70 dark:shadow-xl dark:shadow-cyan-500/40"

// Combined pill badge
className="dark:bg-cyan-500/35 dark:text-cyan-100 dark:border dark:border-cyan-300/60 dark:shadow-lg dark:shadow-cyan-500/50"
```

### Direct RGB Usage
```css
/* For canvas rendering */
ctx.strokeStyle = "rgba(34, 211, 238, 0.95)"; /* cyan-300 bright */
ctx.fillStyle = "rgba(6, 182, 212, 0.35)";   /* cyan-500 glow */
```

## Tailwind Color Scale Reference

### Dark Mode Text Colors
- `dark:text-cyan-300` ← Use this
- `dark:text-cyan-200` ← Too light, less vibrant
- `dark:text-cyan-400` ← Darker, less bright

### Shadow Opacity Ranges
```
Recommended opacity values for glows:
- Text shadow: 0.35-0.50
- Border shadow: 0.40-0.45
- Large container: 0.35-0.50
```

## Testing Your Implementation

### Visual Verification
1. Open app in dark mode
2. Each category section should have distinct neon tint
3. Text should be bright and readable
4. Glows should create depth without blur

### Color Picker Verification
Use developer tools to inspect elements and verify:
```
Expected computed style for section:
background: linear-gradient(135deg, rgba(...), rgba(...), rgba(...))
box-shadow: 0 ... px var(--canvas-[color]-shadow)
```

### Print Test
When printed, colors should be muted (fallback to light mode colors)
